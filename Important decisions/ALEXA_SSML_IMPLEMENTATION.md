# SSML Implementation for Alexa Responses

**Date:** March 13, 2026  
**Sprint:** Ngrok Integration & Security  
**Status:** ✅ Implemented

## Problem

Alexa was reading SSML markup literally instead of interpreting the audio effects:
- `<break time=\"400ms\"/>` was read as text instead of producing a 400ms pause
- `<amazon:effect name=\"whispered\">` was read as-is instead of applying whispered effect
- `<prosody rate=\"85%\">` was not adjusting speech rate

**Root cause:** The response format was sending `"type": "PlainText"` instead of `"type": "SSML"`.

## Solution

Updated all Alexa response generation in `backend/src/index.js`:

### Before (PlainText)
```json
{
  "outputSpeech": {
    "type": "PlainText",
    "text": "Rosa se despierta <break time=\"400ms\"/> en silencio."
  }
}
```

### After (SSML)
```json
{
  "outputSpeech": {
    "type": "SSML",
    "ssml": "<speak>Rosa se despierta <break time=\"400ms\"/> en silencio.</speak>"
  }
}
```

### Implementation Changes

1. **Updated `alexaResponse()` function** (line 385):
   - Changed `outputSpeech.type` from `"PlainText"` to `"SSML"`
   - Changed property from `text` to `ssml`
   - Auto-wrap content in `<speak>...</speak>` tags if not already wrapped

2. **Updated permission request responses** (lines 503, 532, 600):
   - Same pattern: `type: "SSML"` with `ssml` property
   - Ensures consistent SSML handling across all response types

3. **Added auto-wrapping logic:**
   ```js
   const ssmlText = text.trim().startsWith('<speak') ? text : `<speak>${text}</speak>`;
   ```

## Impact

### Beneficial
- ✅ Narrative content now reads with intended audio effects (breaks, prosody, whispered)
- ✅ Improved immersion and emotional engagement in story experience
- ✅ All scene transitions and narration now include pacing via `<break>` tags
- ✅ Character emotions conveyed through prosody adjustments and effects

### Considerations
- SSML tags must be properly formatted; malformed SSML may cause Alexa to fall back to PlainText
- Text content passed to `alexaResponse()` should already contain valid SSML markup (generated by backend/src/prompts.js)
- ngrok tunnel to Alexa Developer Console requires HTTPS (Alexa enforces SSL/TLS)

## Security & Deployment

### For local ngrok testing:

### For production:
- Implement Alexa request signature verification before processing
- Use reserved ngrok domain or migrate to persistent HTTPS hosting
- Validate SSML content server-side to prevent injection attacks
- Add middleware to strip/sanitize untrusted SSML tags if loading content from external sources
